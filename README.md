# AI Technical Director


## Workflow/Flowchart

When a user submits a prompt (e.g., *"Block out a sci-fi scene with our cyborg"*), the agent enters a persistent state-machine loop:

```text
[User Prompt] 
      │
      ▼
[LangGraph Agent Node] ──► Analyzes request and determines required tools.
      │
      ├─► 1. Tool: read_pipeline_docs()
      │      └─ Grabs: Studio naming conventions & output constraints.
      │
      ├─► 2. Tool: search_studio_library()
      │      └─ Grabs: Absolute file paths of required 3D assets.
      │
      ▼
[Agent Drafts Python Script] ──► Writes `bpy` code using picked assets.
      │
      ▼
[ HITL ] ── (UI Locks & waits for Artist's input)
      │
      ├─► [ REJECT ] ──► Injects given Artist feedback ──► (Loops back to Agent Node)
      │
      ▼
[ APPROVE ]
      │
      ▼
[ Security Scanner ] ──► Scans code for malicious imports (os, sys, subprocess).
      │
      ▼
[ 3. Tool: execute_blender_script() ] 
      │      └─ Grabs: The approved Python script and target output filename.
      │      └─ Action: Executes headless Blender via a subprocess
      │
      ▼
[ Final Scene Output & Summary ]

```

---

## Tools

### 1. `read_pipeline_docs`

* **What it grabs:** Parses the local `mock_studio_library/naming_conventions.txt` file.
* **Purpose:** Forces the LLM to ground its file-naming logic in actual studio rules rather than hallucinating naming conventions. Contains failsafe fallback names if the document is missing.

### 2. `search_studio_library`

* **What it grabs:** Takes a list of keywords and recursively walks the `mock_studio_library` directory, returningn matching absolute file paths (limited to 10).
* **Purpose:** Ensures the agent only attempts to spawn assets that physically exist on the studio's server. Protected by `pathlib` to prevent directory traversal outside the authorized library (dummy assets curently).

### 3. `execute_blender_script`

* **What it grabs:** Takes the raw Python code and the `output_filename` generated by the agent.
* **Purpose:** 1. Parses the code into an Abstract Syntax Tree to verify no unauthorized modules are imported.
2. Injects the file-save command.
3. Executes Blender headlessly (`blender -b -P`).
4. Returns the combined `stdout`/`stderr` logs or validates that the resulting `.blend` file was successfully written.

## Installation & Usage


### Setup

1. Clone the repository:
```bash
git clone [https://github.com/Tym154/AI_TA.git](https://github.com/Tym154/AI_TA.git)
cd AT-TA

```


2. Install dependencies:
```bash
pip install -r requirements.txt

```


3. Set your API Key:
Create a `.env` file in the root directory and add:
```env
ANTHROPIC_API_KEY=key_provided_by_anthropic

```


4. Run the application:
```bash
streamlit run app.py

```
